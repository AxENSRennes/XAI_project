# Reproducing "Learning Audio Concepts from Counterfactual Natural Language"

This repository contains the code for our reproduction of the paper
[Learning Audio Concepts from Counterfactual Natural Language (Vosoughi et al., ICASSP 2024)](https://arxiv.org/abs/2401.04935),
produced as part of the **MVA XAI course** (2025-2026) at ENS Paris-Saclay.

**Team 24:** Côme Hosxe (CentraleSupélec), Axel Mauroy (ENS Rennes), Zhen-Jia Xu (ENS Paris-Saclay)

The original paper extends CLAP (Contrastive Language-Audio Pretraining) with two extra loss
terms that use counterfactual captions to make the model more discriminative. This code does not
replicate the authors' private implementation exactly. Instead, it provides a fully runnable
pipeline built from the public release artefacts, with the following differences from the paper:

- We use `laion/clap-htsat-fused` (HTSAT-Swin transformer) instead of PANNs ResNet-38 + CLIP.
- We train only on Clotho (~19,000 pairs) instead of AudioCaps + Clotho + MACS (~91,000 pairs).
- For captions where no LLM counterfactual is available, we apply a rule-based lexical fallback.

Our report and results are in `report.pdf`.

## Setup

Install the dependencies into your virtual environment:

```bash
pip install -r requirements.txt
```

The code uses Hugging Face models. If the global HF cache is read-only on your machine,
set a local cache directory before running any script:

```bash
export HF_HOME=/your/path/.hf-cache
export HF_HUB_CACHE=/your/path/.hf-cache/hub
export TRANSFORMERS_CACHE=/your/path/.hf-cache/transformers
```

## Step-by-step pipeline

### 1. Download the public release JSON files

```bash
python scripts/download_release_assets.py --output-dir data/release_json
```

This downloads the counterfactual annotation JSON files released by the original authors
for Clotho, AudioCaps and MACS.

### 2. Build flat JSONL manifests

Each manifest is a flat JSONL file where every line is one (audio, caption, counterfactual)
triple. You need one manifest for training and one for evaluation.

```bash
# Development set (LLM-generated counterfactuals)
python scripts/build_manifest.py \
  --release-json data/release_json/clotho-development-counterfactual.json \
  --dataset-root /path/to/clotho/audio \
  --output data/manifests/clotho_dev.jsonl \
  --generate-missing-counterfactuals

# Validation set (rule-based counterfactuals, LLM CFs not available for this split)
python scripts/build_manifest.py \
  --release-json data/release_json/clotho-validation-counterfactual.json \
  --dataset-root /path/to/clotho/audio \
  --output data/manifests/clotho_val.jsonl \
  --generate-missing-counterfactuals
```

If some audio files are missing after extraction, use `scripts/filter_manifest.py` to
remove broken rows before training:

```bash
python scripts/filter_manifest.py \
  --input data/manifests/clotho_dev.jsonl \
  --output data/manifests/clotho_dev_clean.jsonl
```

### 3. (Optional) Generate rule-based counterfactuals manually

To test the rule-based fallback on a list of captions:

```bash
python scripts/generate_counterfactuals.py \
  --input examples/factual_captions.txt \
  --output examples/generated_counterfactuals.jsonl
```

### 4. Train

All training configurations are in `configs/`. The main ablation configs are:

| Config file | Description |
|---|---|
| `ablation_no_cf.yaml` | Standard fine-tuning, no counterfactual losses |
| `ablation_angle_only.yaml` | + angle loss ($w_1=1, w_2=0$) |
| `ablation_factual_only.yaml` | + factual loss only ($w_1=0, w_2=100$) |
| `ablation_full.yaml` | Full model ($w_1=1, w_2=100$) |

To reproduce the full model:

```bash
python scripts/train_counterfactual_clap.py \
  --config configs/ablation_full.yaml \
  --train-manifest data/manifests/clotho_dev.jsonl \
  --eval-manifest data/manifests/clotho_val.jsonl \
  --output-dir outputs/ablation_full
```

To run the pre-trained baseline (no fine-tuning, just evaluation):

```bash
python scripts/train_counterfactual_clap.py \
  --config configs/ablation_no_cf.yaml \
  --train-manifest data/manifests/clotho_dev.jsonl \
  --eval-manifest data/manifests/clotho_val.jsonl \
  --output-dir outputs/pretrained_baseline \
  --epochs 0
```

For the $w_2$ sensitivity sweep, use the configs in `configs/sweep_w2_*.yaml`.

### 5. Evaluate

**Text-to-audio retrieval on Clotho:**

```bash
python scripts/evaluate_retrieval.py \
  --checkpoint outputs/ablation_full/best.pt \
  --manifest data/manifests/clotho_val.jsonl
```

**Zero-shot classification on ESC-50:**

```bash
python scripts/evaluate_zero_shot.py \
  --checkpoint outputs/ablation_full/best.pt \
  --manifest data/manifests/esc50_eval.jsonl \
  --label-template "this is a sound of {}"
```

To build the ESC-50 manifest from your audio files:

```bash
python scripts/build_classification_manifest.py \
  --dataset-root /path/to/esc50 \
  --output data/manifests/esc50_eval.jsonl
```

## Key results

| Model | Top-1 T→A | Top-10 T→A | ESC-50 |
|---|---|---|---|
| Pre-trained CLAP (no fine-tuning) | 0.142 | 0.506 | 0.804 |
| Fine-tuned, no CF losses | 0.180 | 0.573 | 0.792 |
| + Factual loss only ($w_2=100$) | 0.184 | 0.579 | — |
| Full model ($w_1=1, w_2=100$) | **0.184** | **0.580** | **0.814** |

The key finding is that the factual consistency loss is the main driver of improvement.
The angle loss alone provides almost no benefit when the backbone is already strong, but
combining both losses gives the best retrieval results and prevents the zero-shot
performance drop caused by standard fine-tuning.

## Known limitations and differences from the paper

- **Backbone:** `laion/clap-htsat-fused` is a much stronger pre-trained model than the
  PANNs ResNet-38 used in the paper. Our absolute numbers are higher, but the relative
  improvement pattern is the same.
- **Training data:** We use Clotho only (~19,000 pairs vs. ~91,000 in the paper). This
  explains most of the difference in absolute values.
- **Validation counterfactuals:** The public Clotho validation JSON contains counterfactuals
  that are identical to the original captions. We apply the rule-based fallback to generate
  usable counterfactuals for evaluation.
- **Batch size:** We use 32. The InfoNCE loss works best with large batches (512+), so our
  contrastive learning is weaker than in the original.
- **Optimal $w_2$:** Our sweep shows that $w_2 = 500$ gives slightly better results than
  $w_2 = 100$ in our setting, likely because a smaller training set needs a stronger
  factual anchor. The main ablation table uses $w_2 = 100$ to match the paper's setting.
